# Study Walkthrough

## Narrative Loop

1. Read the study question.
2. Review the four-gate readiness framework.
3. Inspect the code snippet and linked source.
4. Select a synthetic customer workflow.
5. Run checks to score readiness and expose the current blocker.
6. Repair the blocker so the workflow moves from review to ready.
7. Generate a recommendation with one action and one product signal.
8. Read the learnings to understand what the experiment suggests.

## Design Notes

The page is deliberately more explanatory than the earlier dashboard version. The goal is to make the investigation legible first, then let the controls prove that the study has working implementation underneath it. Each scenario carries enough structure to show the tradeoffs: incomplete data, schema drift, source citations, review gates, and exportable product briefs.
